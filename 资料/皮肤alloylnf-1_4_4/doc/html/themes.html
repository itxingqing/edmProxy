Using Themes with the Alloy Look&Feel

Setting the Theme

There are two different ways to set the current theme. One way is to set the theme when creating the look and feel instance. The class com.incors.plaf.alloy.AlloyLookAndFeel has the constructor AlloyLookAndFeel(AlloyTheme theme) which takes a theme object as an argument. If you are using the empty constructor AlloyLookAndFeel(), the Default Theme will be used.

Setting the theme when creating an instance of AlloyLookAndFeel can look like this:

try {
  com.incors.plaf.alloy.AlloyTheme theme = new com.incors.plaf.alloy.themes.bedouin.BedouinTheme();
  // The lines below can be used for setting the other themes.
  // com.incors.plaf.alloy.AlloyTheme theme = new com.incors.plaf.alloy.themes.glass.GlassTheme();
  // com.incors.plaf.alloy.AlloyTheme theme = new com.incors.plaf.alloy.themes.acid.AcidTheme();
  javax.swing.LookAndFeel alloyLnF = new com.incors.plaf.alloy.AlloyLookAndFeel(theme);        
  javax.swing.UIManager.setLookAndFeel(alloyLnF);
} catch (javax.swing.UnsupportedLookAndFeelException ex) {
  // You may handle the exception here
}


The Alloy Look&Feel allows switching of the theme during runtime. For this purpose the class com.incors.plaf.alloy.AlloyLookAndFeel provides the method setTheme(AlloyTheme newTheme, boolean updateAllUIs). This method makes theme switching very convenient for the fact that it requires only one line of code in most cases. Set the parameter updateAllUIs to true if you want the Alloy Look&Feel to automatically update the UIs of all components in the component hierarchy. If you set the parameter to false, you will have to update the UIs by yourself, which allows update of only part of the component hierarchy.

/*
  Note: the object "alloyLnF" must be the instance of the class "com.incors.plaf.alloy.AlloyLookAndFeel" 
  that is set as the current LookAndFeel.
*/
alloyLnf.setTheme(new com.incors.plaf.alloy.themes.bedouin.BedouinTheme(), true);
// Alternatively use one of the following lines to set the other themes 
//alloyLnf.setTheme(new com.incors.plaf.alloy.themes.glass.GlassTheme(), true);
//alloyLnf.setTheme(new com.incors.plaf.alloy.themes.acid.AcidTheme(), true);
//alloyLnf.setTheme(new com.incors.plaf.alloy.DefaultAlloyTheme(), true);


Please keep in mind the potential issues with "orphan" components and layout corruption which were mentioned in the basic usage guide.

Setting the Theme Property

To set the initially used theme you can also use the theme property. To do this, invoke invoke this line before creating the first instance of the class AlloyLookAndFeel:

AlloyLookAndFeel.setProperty("alloy.theme", "glass");

Possible values for the property are "default", "glass", "acid", and "bedouin".

For more information on setting properties please refer to section about properties.


Creating Custom Themes

Since version 1.2 the Alloy Look&Feel provides a simple way for creating customized themes. For creating a custom theme all you have to do is to call the method createTheme() of the class com.incors.plaf.alloy.themes.custom.CustomThemeFactory.

For creating a custom theme you will have to decide on six colors:
  • Contrast Color: This color will be used for scroll bars, progress bars, and frame decorations. For the overall impression of you theme this color is the most important.
  • Standard Color: This color will be used for panels, buttons, and many other backgrounds. Since most of the user interface will be in this color, you should use a rather light color with no or only slight saturation.
  • Desktop Color: This color is only used for painting the background of a JDesktopPane. In most cases it is a good idea to use a quite dark color with the same hue as the secondary color.
  • Selection Color: This color is used for selection indicators like the dot of a radio button and the hook of a check box. Usually this color is very similar to the primary color.
  • Rollover Color: This color is used for painting the rollover indicator on buttons, radio buttons, and check boxes. Use a quite strong an bright color for this.
  • Highlight Color: This color is used for painting the background of selected text and selected menu items. Often a lighter version of the primary color's complementary color is used for this.

A nice tool for creating your own theme is the WebStart demo application on the Alloy Look&Feel website, which allows interactive creation of custom themes.

Finally here is a code example for creating and setting a custom theme:


// define colors
Color contrast = new Color(171, 177, 128);
Color standard = new Color(242, 242, 235);
 Color desktop = new Color(100, 103, 92);
 Color selection = new Color(112, 142, 148);
 Color rollover = new Color(182, 204, 216);
 Color highlight = new Color(180, 208, 213);

// create custom theme
com.incors.plaf.alloy.AlloyTheme myTheme =
    com.incors.plaf.alloy.themes.custom.CustomThemeFactory.createTheme(contrast,
    standard, desktop, selection, rollover, highlight);

// create look and feel with the custom theme
try {
  javax.swing.LookAndFeel alloyLnF = new com.incors.plaf.alloy.AlloyLookAndFeel(myTheme);
  javax.swing.UIManager.setLookAndFeel(alloyLnF);
} catch (javax.swing.UnsupportedLookAndFeelException ex) {
  // You may handle the exception here
}
